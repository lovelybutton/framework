Forms

Form Usage

Table of Contents:

    The code here represents all markup for standard forms. Each section describes a specific aspect of the form modules, including required and optional classes, as well as considerations for style variations.

    Form Structure

    Types of forms

    All forms begin by being wrapped in a div with a class of form. This div will then need a second class which described what kind of form it is. The available form type classes are:

    (click to view form sample)
    • form.one-column
    • form.one-column.liquid
    • form.one-column.liquid.sidebar
    • form.two-column
    • form.three-column
    • form.cta
    • form.cta.lp

    The structure of the modules differs slightly depending on what type of form is being used, but there are common elements within all forms.

    Form Title

    If a form requires it, a div with a class of form-title can be added before the inner div. Sidebar and CTA forms do not typically require this. In this div you can add an h3 tag containing the form title. On the examples below there is also a border div as well.

    <div class="form-title has-bdr">
	<h3>Please complete the form below</h3>
	<b class="bdr bdr-bottom bdr-dash"></b> 
</div>

    FormBox / ConfBox

    The convention here is unchanged. All forms require at least a div with a class of formBox. If there is custom confirmation content then it will also require a div with a class of confBox. If the form is a demo form, it will also require a success div and a maint div.

    Demo Confirmations

    Demo confirmations specifically have a consistent structure. They contain a list with two steps that has a class of list-steps. The first steps has the login credentials which consist of these elements:

    • div.cred-wrap - This div contains a credential
    • span.cred-title - A span that contains the title for the credential (User or ID Password)
    • div.cred - This wraps the span with the credential (span.demoUsername or span.demoPassword)

    Form cells

    Directly within the <form> tag, a form will contain one or many form-cells. These form cells are the building blocks of your form, and can contain text fields, radio buttons, checkboxes, paragraphs, lists or anything else you need in your form. 

    <div class="form-cell">
	<label>...</label>
	<input ...>
	</div>
	<div class="form-cell">
	<p>...</p>
</div>

    Modifying form cells

    A form-cell has a set width connected to each type of form. For example, a form-cell in a three-column form is 300px wide, causing the form-cells to stack neatly next to each other to fill up a row. If, however, you need a form-cell to span the entire width of your three-column form, add a three-column class, like so:

    <div class="form-cell three-column">
	This form-cell spans the entire width of my form
</div>

    Note that adding a three-column form-cell within a one-column form will not work.

    Section titles

    If you have a subsection that needs a title within the form (for example "Joint account information"), wrap that element in a form-section-title:

    <h4 class="form-section-title">Joint Account Information</div>

    OR

    <label class="for-text label-section-title">Some other section title name</label>

    Labels

    Labels get a specific class, depending on what type of field they're for. The options are:

    • <label class="for-text">
    • <label class="for-select">
    • <label class="for-radio">
    • <label class="for-checkbox">

    All the above labels will get the default font styling. To modify the styles of a specific label, add one of the following classes:

    • "Regular text": txt-regular
    • "Bold text": txt-bold
    <label class="for-text txt-regular" ...>
	This type of style is good for sequential radio buttons
</label>
    <label class="for-text txt-bold" ...>
	Use this style for consent checkbox labels
</label>

    Ocassionally, your label will spill to two lines. If this is the case in a two-column or three-column form, you'll need to add a modifier class to the adjacent form-cells, forcing all the fields in that row to align. Use the following classes:

    • To match a label that takes up two lines: <div class="form-cell match-label-heightx1">
    • To match a label that takes up three lines: <div class="form-cell match-label-heightx2">

    Fields

    Labels get a specific class, depending on what type of field they're for. The options are:

    A standard text field: 

    <input class="field" .../>

    A standard text area: 

    <input class="field textarea" .../>

    A standard select: 

    <select class="field select" .../>

    A standard checkbox (does not require field class): 

    <input class="checkbox" .../>

    A standard radio button (does not require field class): 

    <input class="radio" .../>

    Additional elements

    Scrollable

    A div with a fixed height, border and scrollbar, useful for hiding large blocks of legal text in a small space .

    <div class="form-cell three-column">
	<div class="scrollable">...</div>
</div>

    Instruction

    Small note or message that appears beneath an input field.

    <div class="form-cell">
	<label>...</label>
	<input ...>
	<div class="instruction"> ... </div>
</div>

    Provider info

    Structure that appears on lod forms ocassionally.

    <div class="form-cell three-column providerInfo">
	<div class="clearfix">
	<div class="cat">Category:</div>
	<div class="val">Value</div>
	</div>
</div>
